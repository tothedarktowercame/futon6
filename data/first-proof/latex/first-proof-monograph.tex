\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
This monograph reports a multi-agent proof sprint on ten research-level
problems, combining rapid draft generation with adversarial verification,
targeted repair, and explicit provenance. The workflow uses wiring-diagram
decompositions of claim dependencies to localize gaps and coordinate
reviewer-driven revisions. Final outcomes are heterogeneous but explicit:
the manuscript distinguishes \emph{mathematical status} from
\emph{QC-validation status}. In the current ledger, Problems~1, 2, 3, and~8
are treated as mathematically closed with validation-complete records
(Problem~2 still depends on the cited newvector/test-vector theorem input);
Problems~5 and~7 are mathematically closed with partial QC status (scope-limited
closure for Problem~5, and an independent theorem-ledger re-check pending for
Problem~7). Problem~4 is partial: proved analytically for \(n \le 3\), and
proved for \(n=4\) via the Path~2 algebraic certificate, with higher \(n\)
open. Problem~6 is partial: the \(K_n\) case is proved with \(c_0 = 1/3\), while
the general-graph bridge remains open (formal \(\bar d < 1\) bound at
\(M_t \ne 0\)). Problem~9 has strong computational evidence but lacks a clean
algebraic closure of the universal converse step. Problem~10 remains
conditional under explicitly stated assumptions. The main methodological result
is that structure-aware verification and layer-switching strategies improve
reliability and calibration in compressed proof sprints.

\chapter*{Reading Convention}
\addcontentsline{toc}{chapter}{Reading Convention}

This manuscript uses boxed annotations selectively. In the current revision,
the full proof chapters are primarily unboxed; process boxes appear only where
there is substantive process context to preserve.

\begin{itemize}[leftmargin=1.5em]
\item \textbf{Unboxed text} is the proof proper. A reader following the
  five-page-per-problem brief can read only the unboxed text and obtain a
  self-contained argument (or, where an argument is incomplete, a clearly
  stated conditional result).
\item \textbf{Status Note boxes} appear in the annotated overview chapters to
  summarize closure/QC state for each problem.
\item \textbf{Process Note boxes} appear in process-facing parts (for example,
  prompt excerpts) when methodological scope or reviewer context is needed.
\item Dead-path / revision / open-obligation semantics are maintained in the
  tooling vocabulary, but are not instantiated as standalone boxed blocks in
  the current full-proof chapter text.
\end{itemize}
